概述

  • Spring是一个轻量级的java企业级应用的开源开发框架。
  • Spring系列常用的5个框架有，SpringFramework、SpringBoot、SpringData、SpringCloud、SpringSecurity。
  • Spring的核心是控制反转（IoC）和面向切面（AOP）。
  • Ioc即控制反转，通过Spring提供的Ioc容器，我们可以将对象之间的依赖关系交由Spring进行控制，在一定程度上促进了低耦合，简化了开发。
  • AOP即面向切面编程，Spring支持AOP，将程序中的独立于其他功能的方法抽取出来，比如事务，日志，利用动态代理的方式，在不修改源代码的情况下，对方法进行功能的增强。
  • Spring还支持事务管理以及方便集成其他各种优秀的框架。

框架

Ioc原理

传统的方式：通过new关键字主动创建一个对象。
IoC方式：对象的生命周期由Spring来管理，直接从Spring那里获取一个对象。

依赖注入

两种方式：

  • 基于xml装配
    • 基于构造方法
    • 基于setter方法
  • 基于注解装配

基于xml装配

基于构造方法

user.java

public class User {

    private Integer uid;
    private String username;
    private Integer age;

    public User(Integer uid, String username) {
        super();
        this.uid = uid;
        this.username = username;
    }

    public User(String username, Integer age) {
        super();
        this.username = username;
        this.age = age;
    }
}

applicationContext.xml

<!-- 构造方法注入 
    * <constructor-arg> 用于配置构造方法一个参数argument
        name ：参数的名称
        value：设置普通数据
        ref：引用数据，一般是另一个bean id值

        index ：参数的索引号，从0开始 。如果只有索引，匹配到了多个构造方法时，默认使用第一个。
        type ：确定参数类型
    例如：使用名称name
        <constructor-arg name="username" value="jack"></constructor-arg>
        <constructor-arg name="age" value="18"></constructor-arg>
    例如2：【类型type 和  索引 index】
        <constructor-arg index="0" type="java.lang.String" value="1"></constructor-arg>
        <constructor-arg index="1" type="java.lang.Integer" value="2"></constructor-arg>
-->
<bean id="userId" class="/.User" >
    <constructor-arg index="0" type="java.lang.String" value="1"></constructor-arg>
    <constructor-arg index="1" type="java.lang.Integer" value="2"></constructor-arg>
</bean>
基于setter方法

applicationContext.xml

<!-- setter方法注入 
    * 普通数据 
        <property name="" value="值">
        等效写法
        <property name="">
            <value>值</value>
        </property>
    * 引用数据
        <property name="" ref="另一个bean">
        等效写法
        <property name="">
            <ref bean="另一个bean"/>
        </property>

-->
<bean id="personId" class="Person">
    <property name="pname" value="chasen"></property>
    <property name="age">
        <value>1234</value>
    </property>

    <property name="homeAddr" ref="homeAddrId"></property>
    <property name="companyAddr">
        <ref bean="companyAddrId"/>
    </property>
</bean>

基于注解装配

注解，就是一个类。
开发中，使用注解取代xml配之文件
@Component取代<bean class="">
@Component("id") 取代<bean id="" class="">

面向切面AOP

  • 功能分两大类，辅助功能和核心业务功能
  • 辅助功能和核心业务功能彼此独立进行开发
  • 比如登陆功能，即便是没有性能统计和日志输出，也可以正常运行
  • 如果有需要，就把”日志输出” 功能和 “登陆” 功能 编织在一起，这样登陆的时候，就可以看到日志输出了
  • 辅助功能，又叫做切面，这种能够选择性的，低耦合的把切面和核心业务功能结合在一起的编程思想，就叫做切面编程
