java线程的暂停和恢复

发表于 2018-10-26 | 分类于 编程 |

线程状态

基础前提

  • Java中的每个对象都有一个监视器，来监测并发代码的重入。
  • synchronized(obj):对obj加上同步锁并获取该obj的监视器（monitor）。
  • synchronized(this){}等同于public synchronized void method(){}。
  • 当想要调用wait()进行线程等待时，必须要取得这个锁对象的控制权（监视器），一般是放到synchronized(obj)代码中。
  • 在while循环里而不是if语句下使用wait，这样，会在线程暂停恢复后都检查wait的条件，并在条件实际上并未改变的情况下处理唤醒通知
  • notify()方法只会通知等待队列中的第一个相关线程（不会通知优先级比较高的线程）。
  • notifyAll()通知所有等待该竞争资源的线程（也不会按照线程的优先级来执行）。
